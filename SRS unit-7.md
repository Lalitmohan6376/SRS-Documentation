# 📘 Unit 7 — Software Testing & Quality Assurance (SQA)

---

# 🌟 Introduction

Software Testing and Quality Assurance (SQA) is a very important phase in software development 🧪💻

It ensures that the software:

* ⚙️ works correctly
* 🐞 has minimum bugs
* 🔒 is secure
* ⚡ performs well
* 👥 meets user requirements

👉 Testing is done to make sure the final product is reliable and stable.

---

# 🧠 What is Software Testing?

Software Testing is:

> The process of checking whether the software works as expected or not 🧪

It helps in finding errors, bugs, and issues before release.

---

# 🎯 Goals of Software Testing

Testing helps in:

* 🐞 Finding bugs
* ✅ Verifying requirements
* 🔒 Ensuring security
* ⚡ Improving performance
* 👥 Enhancing user experience

---

# 🧠 What is Quality Assurance (QA)?

Quality Assurance is:

> A process to ensure that the software development process produces high-quality software 📊

👉 QA focuses on **process improvement**, not just testing.

---

# 🔄 Difference Between Testing and QA

| 🧪 Testing             | 📊 QA                   |
| ---------------------- | ----------------------- |
| Finds bugs             | Prevents bugs           |
| Product-focused        | Process-focused         |
| Done after development | Done throughout SDLC    |
| Ensures correctness    | Ensures quality process |

---

# 🧪 Types of Software Testing

---

## 🔹 1. Unit Testing

Testing individual components or functions.

### Example:

Testing login function only 🔐

---

## 🔗 2. Integration Testing

Testing combined modules.

### Example:

Login + Dashboard working together 👥

---

## 🌐 3. System Testing

Testing the entire system as a whole.

---

## 👤 4. User Acceptance Testing (UAT)

Final testing by real users.

### Example:

Client checks if software meets requirements ✔️

---

## ⚡ 5. Performance Testing

Checks speed and stability.

### Includes:

* Load testing 📈
* Stress testing 🔥

---

## 🔒 6. Security Testing

Checks system protection.

### Example:

* SQL injection testing
* Password encryption check

---

## 📱 7. Usability Testing

Checks how easy the software is to use 👥

---

# 🐞 Types of Bugs

* ❌ Syntax errors
* ⚠️ Logical errors
* 🧩 Functional errors
* 🔒 Security bugs
* ⚡ Performance issues

---

# 🧪 Testing Techniques

---

## ⚫ Black Box Testing

* No knowledge of code required
* Focus on input and output

### Example:

Login system testing 🔐

---

## ⚪ White Box Testing

* Code is tested internally
* Developers check logic

---

## 🔘 Grey Box Testing

* Combination of black + white box testing

---

---

## 🔒 3. Security

# 🔄 Testing Life Cycle (STLC)

Software Testing Life Cycle includes:

1. 📋 Requirement Analysis
2. 🧪 Test Planning
3. 🎨 Test Case Design
4. ⚙️ Test Execution
5. 🐞 Bug Reporting
6. 🔁 Retesting
7. 📊 Test Closure
