# 📘 Introduction to SRS

---

## 🚀 What is SRS?

SRS (Software Requirements Specification) is a document that clearly explains what a software system should do and how it should behave.

It is created before starting the development of any project.

---

## 🧠 Simple Meaning

SRS is like a **blueprint** of a project.

Before building anything, we first decide:
- What features are needed
- How the system should perform

---

## ❓ Why SRS is Important

- It helps in understanding the project clearly  
- It avoids confusion between developer and client  
- It saves time and effort during development  
- It makes the project structured and organized  

---

## 💡 Real-Life Example

Imagine you want to build a **Movie Recommendation System**.

Before coding, you decide:
- User can login  
- System recommends movies  
- App should be fast  

All these points are written in SRS.

---

## 🎯 Key Idea

SRS answers two main questions:

- What should the system do?  
- How should the system behave?  

---

## ⚡ Summary

SRS is the first and most important step in building any software project.
