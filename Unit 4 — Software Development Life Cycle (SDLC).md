# 📘 Unit 4 — Software Development Life Cycle (SDLC)

---

## 🌟 Introduction

Software Development Life Cycle (SDLC) is a structured process used to develop software systems step by step 🏗️💻

It helps teams:

* 📌 plan the project
* ⚙️ develop software properly
* 🧪 test the system
* 🚀 deploy successfully
* 🔧 maintain the application

Without SDLC, software projects can become disorganized and difficult to manage ❌

---

# 🧠 What is SDLC?

SDLC is a process that defines:

> How software is planned, built, tested, and maintained 📄⚙️

It provides a proper workflow for software development.

---

# 🎯 Goals of SDLC

SDLC helps in:

* ✅ Improving software quality
* ⏳ Completing projects on time
* 💰 Reducing development cost
* 🐞 Reducing bugs and errors
* 📈 Improving scalability and maintenance

---

# 🔄 Phases of SDLC

SDLC mainly contains the following phases:

1. 📋 Planning
2. 📑 Requirement Analysis
3. 🏗️ System Design
4. 👨‍💻 Development
5. 🧪 Testing
6. 🚀 Deployment
7. 🔧 Maintenance

---

# 📋 1. Planning Phase

This is the first phase of SDLC.

In this phase, teams decide:

* 🎯 Project goals
* 💰 Budget
* ⏳ Timeline
* 👨‍💻 Team members
* ⚠️ Risks

### 📌 Example:

Building an AI Interview Platform.

Planning includes:

* required technologies
* estimated cost
* development timeline

---

# 📑 2. Requirement Analysis Phase

In this phase:

* 📥 requirements are collected
* 🔍 requirements are analyzed
* 📄 SRS document is created

### Includes:

* Functional requirements ⚙️
* Non-functional requirements 🚀
* User needs 👥

👉 This phase decides what the software should do.

---

# 🏗️ 3. System Design Phase

System design defines:

> How the system will be built ⚙️

### Includes:

* 🗄️ Database design
* 🔌 API structure
* 🧩 System architecture
* 🎨 UI/UX design

### Example:

Using:

* React ⚛️
* FastAPI ⚡
* PostgreSQL 🗄️

---

# 👨‍💻 4. Development Phase

Actual coding happens in this phase 💻

Developers:

* write code
* build features
* connect database
* create APIs

### Technologies Example:

* Python 🐍
* JavaScript ⚡
* React ⚛️
* Node.js 🌐

---

# 🧪 5. Testing Phase

Testing checks whether the system works properly ✅

### Purpose:

* 🐞 Find bugs
* ⚠️ Detect errors
* 🔒 Check security
* ⚡ Test performance

---

## 🔍 Types of Testing

### ✅ Unit Testing

Testing small parts of code.

### 🔗 Integration Testing

Testing connected modules together.

### 🌐 System Testing

Testing the complete application.

### 👤 User Acceptance Testing (UAT)

Testing by real users or clients.

---

# 🚀 6. Deployment Phase

Deployment means:

> Making the software live for users 🌍

### Examples:

* Deploying website on cloud ☁️
* Publishing mobile app 📱
* Launching APIs 🔌

### Popular Platforms:

* AWS ☁️
* Vercel ▲
* Netlify 🌐
* Render 🚀

---

# 🔧 7. Maintenance Phase

After deployment, software still needs updates 🔄

### Maintenance Includes:

* 🐞 Bug fixing
* ⚡ Performance improvements
* 🔒 Security updates
* ➕ Adding new features

👉 Maintenance is a continuous process.

---

# 📊 Popular SDLC Models

Different projects use different SDLC models.

---

## 🌊 1. Waterfall Model

A linear step-by-step model.

### Flow:

Planning → Design → Development → Testing → Deployment

### ✅ Advantages:

* Simple
* Easy to understand

### ❌ Disadvantages:

* Difficult to change requirements later

---

## 🔄 2. Agile Model

Development happens in small iterations called sprints ⚡

### Features:

* Continuous feedback
* Fast development
* Flexible requirements

### Used In:

* startups 🚀
* modern software companies 💻

---

## 🌀 3. Spiral Model

Combines:

* planning
* risk analysis
* development

### Best For:

* Large projects
* High-risk systems

---

## 🧩 4. Incremental Model

Software is developed in smaller parts/modules.

### Example:

* First login system
* Then dashboard
* Then AI features

---

# 👥 Role of Team Members in SDLC

### 📋 Project Manager

Manages project and team.

### 🧠 Business Analyst

Understands requirements.

### 👨‍💻 Developers

Write code.

### 🧪 Testers

Test application.

### 🎨 UI/UX Designers

Design interfaces.

### 🔧 DevOps Engineers

Handle deployment and servers.

---

# ⚠️ Common Problems in SDLC

* ❌ Poor planning
* 📄 Unclear requirements
* ⏳ Delayed testing
* 🔒 Ignoring security
* 🚫 Lack of communication
